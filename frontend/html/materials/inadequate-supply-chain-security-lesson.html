Inadequate Supply Chain Security

The software supply chain involves many parties contributing to application development. Risks arise when any element in the chain is insecure.

 

Examples of Supply Chain Risks

 

  • Using dependencies that are no longer supported
  • Downloading libraries from untrusted sources
  • Not verifying integrity of third-party components
  • Lack of security audits on vendors

 

Risk Mitigation Strategies

 

  • Use dependency managers with security validation
  • Conduct regular security audits
  • Use tools like Software Composition Analysis (SCA)
  • Ensure all third-party components have clear and secure licenses
