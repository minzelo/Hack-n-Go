Improper Credential Usage

Improper Credential Usage is a common mistake in storing, sharing, or using credentials insecurely, such as storing passwords in source code, using default credentials, or sharing credentials via email.

 

Examples of Improper Credential Usage

 

  • Storing credentials directly in code (hardcoded)
  • Using default credentials (admin/admin)
  • Storing configuration files without encryption
  • Sending passwords via email

 

Secure Credential Management

 

  • Use password managers
  • Implement two-factor authentication (2FA)
  • Use environment variables and credential vaults
  • Encrypt all credentials during storage

Remember, proper credential management is essential for maintaining security. Never store sensitive information in plain text and always use secure methods for credential storage and transmission.
